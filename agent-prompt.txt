## User Story
As a user, I want dedicated piano roll tools for draw, paint, select, and erase with `1-4` shortcuts, so that I can edit MIDI faster without mode confusion.

## Source
Unchecked item from `docs/design/UX_IMPROVEMENT_CHECKLIST.md`:
- P1 Piano Roll: **Draw/paint tools**

## Problem
The checklist identifies missing tool modes in the piano roll. Without explicit tools and shortcuts, MIDI editing is slower and less discoverable.

## Scope
Add tool-state support for pencil, paint, select, and erase in the piano roll UI and state layer.

## Acceptance Criteria
- Piano roll exposes four tools: pencil, paint, select, erase.
- Number keys `1`, `2`, `3`, `4` switch tools reliably when piano roll focus is active.
- Pencil creates a single note, paint repeats notes across grid steps, select supports note selection, and erase removes targeted notes.
- Tool changes use Zustand-backed state so the same behavior is reachable from `window.__store`.
- Toolbar buttons have ARIA labels and visible active-state feedback.
- Browser testing covers a short composition workflow using all four tools.
