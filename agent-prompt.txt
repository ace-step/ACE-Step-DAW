## User Story
As a user, I want prompt autocomplete suggestions while typing, so that I can build useful generation prompts faster and discover supported musical descriptors.

## Problem
Prompt autocomplete is called out in the UX checklist, but it is not yet tracked as a standalone implementation task.

## Scope
- Suggest genres, instruments, moods, and production techniques as the user types.
- Support keyboard navigation and mouse selection.
- Rank suggestions by prefix/fuzzy relevance.
- Keep the suggestion model usable from both the UI and agent-facing APIs.

## Acceptance Criteria
- Typing in the prompt field opens relevant autocomplete suggestions.
- Arrow keys + Enter can select a suggestion without leaving the keyboard.
- Suggestions do not interfere with normal text editing or IME input.
- Suggestion state is testable through accessible DOM and/or store state.

## Verification
- `npm run build`
- Browser test: type partial prompt tokens such as `lof`, `warm`, or `analog` and verify suggestions appear and insert correctly.
- Agent test: drive the prompt input through automation and confirm autocomplete options are discoverable via accessibility labels.
