IMPLEMENT ISSUE #461: feat: add zoom-to-selection and fit-project shortcuts
## User Story
As a user, I want `Z` to zoom to my current clip or region selection and `Shift+Z` to fit the full project, so that I can move between detailed editing and full-song context instantly.

## Source
Unchecked item from `docs/design/UX_IMPROVEMENT_CHECKLIST.md`:
- Timeline & Navigation
- Zoom to selection

## Problem
The checklist calls for fast navigation shortcuts, but there is no dedicated zoom-to-selection / fit-project workflow tracked as a developer task.

## Scope
- Add `Z` to zoom horizontally to the selected clips or selected region.
- Add `Shift+Z` to fit the full project in view.
- Define fallback behavior when nothing is selected.
- Keep behavior available through the same Zustand action path used by keyboard shortcuts and future agent automation.
- Add ARIA-visible affordances or command registration if the app exposes shortcuts in UI.

## Acceptance Criteria
- Pressing `Z` with a valid selection updates the visible timeline range to fit the selection with small padding.
- Pressing `Shift+Z` fits the full project contents in view.
- With no selection, `Z` uses a documented fallback and does not fail silently.
- The action is reachable through the shared store/action API, not only DOM event handlers.
- No regression to existing zoom gestures.

## Verification
- `npx tsc --noEmit`
- `npm run build`
- Manual workflow: create clips across multiple bars, select a subset, press `Z`, verify the viewport fits the selection; press `Shift+Z`, verify the full arrangement fits again.
- Agent workflow: trigger the same action through `window.__store` or the shared action layer and confirm viewport state changes.
